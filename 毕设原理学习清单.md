## 学习方法

每个知识点，问自己三个问题：
1. **这是什么？** （概念）
2. **为什么要用它？** （解决什么问题）
3. **在我的项目里怎么体现的？** （结合代码讲）

## 面试核心问题

能流畅回答这个问题就成功一半：

> **"介绍一下你的毕设项目。"**

**参考答案**：
```
我的毕设是高校外聘教师管理系统，用于管理外聘老师的招募、授课、考核和薪酬。

技术栈是前后端分离：
- 后端用 Spring Boot + MyBatis-Plus + MySQL
- 前端用 Vue 3 + Element Plus
- 认证用 JWT，缓存用 Redis

主要功能模块有：
1. 外聘老师档案管理
2. 招募公告和多级审批流程
3. 课程安排和授课监督
4. 考核评估自动汇总
5. 薪酬自动计算和结算

开发过程中最大的挑战是多级审批流程的状态管理，
我用状态机的思想，定义了审批节点和流转规则，最终解决了这个问题。

如果重新做，我会考虑加入工作流引擎（如Activiti）来管理审批，
会更灵活、更规范。
```
