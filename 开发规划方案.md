### 系统角色
1. **系统管理员** - 系统配置、权限管理
2. **人事处管理员** - 全局管理、审批、薪酬配置
3. **二级学院管理员** - 本院外聘老师管理、课程安排
4. **外聘老师** - 个人信息、课程查看、薪酬查询
5. **学生** - 课程评价（可选）

## 四、功能模块拆分与开发顺序

### 开发阶段划分

#### 第一阶段：基础框架搭建
1. 项目初始化（Spring Boot + Vue项目结构）
2. 数据库设计与建表
3. 统一响应封装、异常处理
4. 用户认证与权限管理（Spring Security + JWT）
5. 文件上传下载功能

#### 第二阶段：核心业务模块
按依赖关系顺序开发：

```
1. 基础信息管理模块
   ├── 二级学院管理 (CRUD)
   ├── 岗位信息管理 (CRUD)
   └── 外聘老师档案管理 (CRUD + 文件上传)

2. 招募与聘用管理模块
   ├── 招募公告发布
   ├── 在线报名申请
   ├── 材料审核（多轮筛选）
   ├── 聘用审批流程（多级审批）
   └── 电子合同生成与签署

3. 授课管理模块
   ├── 课程安排录入
   ├── 授课计划生成
   ├── 授课监督记录
   ├── 学生评价功能
   └── 调课/停课申请
```

#### 第三阶段：考核与薪酬
```
4. 考核评估模块
   ├── 考核维度配置
   ├── 考核数据自动汇总
   ├── 考核报告生成
   └── 考核结果应用（续聘/解聘触发）

5. 薪酬结算模块
   ├── 薪酬规则配置
   ├── 薪酬自动核算
   ├── 薪酬校验与发放
   └── 历史薪酬查询
```

#### 第四阶段：续聘解聘与统计
```
6. 续聘与解聘管理模块
   ├── 续聘提醒与申请
   ├── 解聘流程处理
   ├── 离职交接管理
   └── 外聘老师状态跟踪

7. 数据统计与分析模块
   ├── 核心数据统计（ECharts可视化）
   ├── 效率分析报表
   └── 数据导出功能

8. 消息与档案管理模块
   ├── 消息通知（站内信/邮件）
   ├── 档案全生命周期管理
   └── 操作日志审计
```

## 六、核心功能实现要点

### 1. 多级审批流程
- 使用状态机模式管理审批状态
- 每个审批节点记录审批人、时间、意见
- 支持驳回并重新提交

### 2. 薪酬自动计算
```
薪酬 = 基础课时费 × 实际课时 × 职称系数 × 考核系数 - 扣款项

其中：
- 职称系数: 助教1.0, 讲师1.1, 副教授1.15, 教授1.2
- 考核系数: 优秀1.1, 合格1.0, 不合格0.8
```

### 3. 考核评分汇总
- 从授课监督表、学生评价表自动获取数据
- 按配置的权重计算综合得分
- 自动判定等级（优秀≥90, 合格≥60, 不合格<60）

### 4. 消息通知机制
- 关键节点自动触发通知
- 支持站内信 + 邮件双通道
- 使用消息队列异步发送

## 八、开发建议

1. **先搭框架**: 使用RuoYi-Vue快速搭建基础框架
2. **核心优先**: 优先完成基础信息管理、招募聘用、授课管理三个核心模块
3. **迭代开发**: 每个模块完成后进行测试，再开发下一个
4. **文档同步**: 边开发边写毕业论文，记录设计思路和实现细节
5. **功能取舍**: 如时间紧张，可简化部分功能（如学生评价可做简化版）
